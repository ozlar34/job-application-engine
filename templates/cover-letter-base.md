# Cover Letter Base Template

> **Synthetic showcase data.** Every fact, number, company, and program below is invented for the fictional persona **Alex Rivera** (see `cv-master.md` / repo README). It demonstrates the *structure* of the cover-letter doctrine the `/tailor-cover-letter` skill consumes — not a real letter. Replace the persona scope and pool entries with your own defensible facts to use the system.

> Source template for every application. Tailor paragraph 1 (hook) and paragraph 3 (transition) per company; leave paragraph 2 (credibility) and paragraph 4 (close) largely intact.
>
> **Rules:**
> - **Length** depends on format. Prose-only letters: 250-350 words (target band; ≤300 sweet spot, 350 hard cap). Structured letters using the patterns below (numbered JD-map, honest-gap, category-fluency): 450-550 words; cap on **reading time, not words** — target ≤60 seconds. Visual structure offsets length.
> - Every load-bearing claim must be a **number, tool name, program name, or named property**. Adjectives ("strong", "deep", "passionate") are deletable; if a sentence collapses when you remove its adjective, it was not earning its place.
> - No em dashes. Use commas, periods, parens, or line breaks.
> - Lead with community / events / creator-ops. AI is one supporting clause, never the spine.
> - Pick a **credibility variant** (A / B / C / D below) to match the role shape. The esports league (SPL) is one strand, not the whole story; do not default to the esports-heavy framing for non-esports roles. **SPL is optional in Variant A** when the target has no gaming/esports/creator-platform overlap (AI/SaaS/no-code/dev-tooling) — the franchise + Creator Ambassador Program + intelligence pipeline + AI centralization carry the credibility on their own.
> - When the league is named, use "global esports league" on first mention, "SPL" on second.
> - **Structural patterns** (numbered JD-map, honest-gap, category-fluency) are the **default layout for all variants** when the JD has enough named strands to map. Prose-only letters (Hook → Credibility → Transition → Close) are the fallback when the JD is too generic to build rows from. See "Structural patterns" section.

## Credibility variant — which to pick

| Target role shape | Variant | Emphasis |
|---|---|---|
| AI-forward, creator platforms, community-lead roles (AI-native and creator-platform companies) | **A — Broad** | Current PM scope + community intel + AI centralization; SPL named once in a subordinate clause (or omitted entirely for non-gaming targets — see SPL-optional rule below) |
| Esports, gaming-native, live-ops-heavy roles (gaming studios, pure-esports seats) | **B — SPL-forward** | SPL scope leads (viewers, regions, languages, event production, creator program); broader ops named briefly |
| Ops / PM / program-manager roles where process-at-scale beats community-at-scale (generic PM seats at any target) | **C — Ops/PM-forward** | PM promotion + tool migration + SOP + AI toolchain centralization lead; community scope named as foundation |
| Highly-specific JDs that name 2-3 candidate strands directly by program name, plus a category Alex can speak to with insider fluency (no-code / AI-builder platforms and seats) | **D — Structured** | **Lede quotes the JD's own vocabulary** ("the first role I've seen where X, Y, Z are named as one mandate"). ## Tailoring checklist (per application)

Before sending, confirm:

- [ ] **Step 0 — Personality depth check.** Open the company research note (external tracker). Is there a `## Personality Hooks` section in the note body?
  - **Yes** → pick 1-2 hooks from the ranked list. Insert at the lede (strongest hook) and/or the closer (secondary hook). **Never in the three-things block** — that's tactical JD-evidence, hooks are strategic framing.
  - **No** → run `/deep-research <Company> for job-search` first. The explicit `for job-search` suffix guarantees the Personality Brief fires regardless of company tier (founders + voice, funding round narrative, customers, category vocabulary). The skill auto-synthesizes ranked hooks and appends them to the research note. Then return here.
  - **Why this exists:** Without personality hooks, every cover letter reads like every other applicant's — competent JD-to-CV mapping but no signal that you actually read past the JD. Applies to **every company**, regardless of how it was discovered or what tier it sits at. The freshly-discovered companies are exactly where personality research delivers the biggest delta.
- [ ] Hiring manager name used if findable (LinkedIn / JD / team page); otherwise "Hiring Team at [Company]"
- [ ] Paragraph 1 hook references one concrete, specific thing (not "your mission")
- [ ] Paragraph 3 transition names the role's unlock and one transferable thread *(prose-only letters)*
- [ ] Credibility variant chosen (A / B / C / D) matches the role shape — SPL-forward framing only on esports/gaming roles; Variant D for highly-specific JDs that name 2-3 strands by program name
- [ ] Positioning variant (A / B / C in positioning-statement.md) matches the credibility choice
- [ ] No em dashes anywhere in the body
- [ ] No "excited about AI" or "passionate about AI" phrasing — and no "passionate about [category]" anywhere
- [ ] **Length gate:** prose-only letters in 250-350 word range; structured letters (Variant D + any optional pattern) 450-550 words / under 60s reading time
- [ ] **Adjective check:** every load-bearing claim contains a number, tool name, program name, or named property; if removing the adjective from a sentence makes it disappear, the sentence was not earning its place
- [ ] SPL phrased as "global esports league" on first mention *(only when SPL is named at all — Variant D and AI/SaaS/no-code Variant A omit SPL entirely)*
- [ ] Numbers pulled from cv-master.md (Locked numbers registry), not from memory
- [ ] **If using Pattern 1 (three-things JD-map):** exactly three rows, headings echo JD vocabulary verbatim, each body has at least one number/tool/program, no body exceeds 60 words
- [ ] **If using Pattern 2 (honest-gap):** the gap is real and visible (not a softball), paired with concrete remediation, three sentences max, one gap only
- [ ] **If using Pattern 3 (category-fluency):** 3-5 reference points named (people, publications, competing products), live tension acknowledged, products personally used on each side of the line, no "passionate" / "excited" verbs

## Structural patterns

> These three patterns are the **default layout** for all variants (A/B/C/D) when the JD has enough material. Each one earns its space by replacing weaker prose with denser, more legible structure. Prose-only is the fallback for JDs too generic to build rows from.

### Pattern 1 — Numbered three-things JD-map

**When to use:** the JD lists 2-3 responsibilities or programs that map directly onto distinct strands of Alex's current scope. Replaces a prose paragraph that strains to do too much.

**Why it works:** breaks one overloaded sentence into 3 atomic claim-rows. Each row is its own promise: a JD-fragment heading + a 2-3 sentence body that proves the claim with a number, tool name, or program name. Reads in scanline mode for a recruiter; reads as evidence for the hiring manager.

**Structure:** a brief lead-in line ("Three things from the JD map cleanly onto what I already do:"), then 3 numbered rows. Each row = `01` / heading / body.

**Hard rules:**
- **Exactly three.** Two looks light; four blurs the message. If the JD only gives you two strong matches, use the honest-gap pattern for slot 3 or fall back to prose.
- **Headings echo the JD's vocabulary**, not yours. If the JD says "Ambassador and Expert programs", the heading is "Ambassador and Expert programs" — not "Creator program".
- **Bodies prove, not assert.** Every body must contain at least one number, named tool, named program, or named property. Adjectives ("strong", "deep", "scalable") are deletable.
- **Do not exceed 60 words per body.** If a body needs more, you are stretching the match — drop the row.

**HTML pattern (matches the cover-letter template tokens):**

```html
<div class="callout">
  <p class="callout-intro">Three things from the JD map cleanly onto what I already do:</p>
  <div class="three">
    <div class="three-item">
      <div class="three-num">01</div>
      <div class="three-head">[JD-vocabulary heading 1]</div>
      <div class="three-body">[Proof body — number, tool, program]</div>
    </div>
    <div class="three-item">
      <div class="three-num">02</div>
      <div class="three-head">[JD-vocabulary heading 2]</div>
      <div class="three-body">[Proof body]</div>
    </div>
    <div class="three-item">
      <div class="three-num">03</div>
      <div class="three-head">[JD-vocabulary heading 3]</div>
      <div class="three-body">[Proof body]</div>
    </div>
  </div>
</div>
```

**Canonical example:** see the Lumen AI cover letter (`applications/2026-04-25-lumen-ai/Alex_Rivera_Cover_Letter_LumenAI.html`), rows 01 (primary community channel ownership), 02 (Ambassador and Expert programs), 03 (B2C and B2B experimentation).

### Pattern 2 — Honest-gap callout

**When to use:** there is a single obvious deficit a hiring manager would notice on first read (e.g., "no production app shipped on the target's own platform", "no published content in the target's primary market language for a single-market seat", "never done in-person events at the target's scale"). Naming it once, with a concrete remediation, is stronger than hoping it goes unnoticed.

**Why it works:** disarms the objection before it forms. Signals self-awareness, not weakness — and the remediation line ("running a trial this week to ship a small internal app") converts a defensive admission into forward motion.

**Hard rules:**
- **One gap, one paragraph, three sentences max.** This is a callout, not a confessional. Naming two gaps reads as overcorrecting and erodes credibility.
- **The gap must be real and visible.** Do not invent a softball ("I'm new to working remotely") to look humble. Pick the deficit a sharp reader would already be circling in red pen.
- **Always pair with a concrete remediation.** Either an action already in motion ("running a trial this week"), a transferable proxy ("the same mechanism applied to a different audience"), or an explicit "happy to talk through how I'd close that on the call." Never just admit the gap and leave it dangling.
- **Never use the gap pattern for non-deficits.** Salary expectations, location, notice period, visa status — these are logistics, not gaps. Logistics belong in the closing paragraph or the cover letter is over-engineered.

**HTML pattern:**

```html
<div class="gap-row">
  <span class="gap-lbl">The honest gap</span>
  [One sentence naming the gap.] [One sentence with concrete remediation — action in motion, transferable proxy, or call-side commitment.]
</div>
```

**Canonical example:** Lumen AI letter, "I have not yet shipped a production app on Lumen AI itself. I am running a trial this week to build a small internal tool so we are working from the same reference point if we get to a call."

### Pattern 3 — Category-fluency paragraph

**When to use:** the target lives in a market category with its own vocabulary, factions, or recurring debates (no-code vs. AI codegen, open-weights vs. closed-source, MLOps vs. AI engineering, creator economy tooling vs. legacy talent agencies, prosumer audio plugins vs. DAW-native tooling). One paragraph that names the right people, sites, products, and the live tension marks Alex as a category insider, not a generic applicant.

**Why it works:** "passionate about no-code" is unverifiable noise. "I follow [category thought leaders] and the [category feed], and I have a working view on where [AI-codegen tools] earn their keep against guardrailed platforms like [no-code builder]" is unfalsifiable AI-speak only if you can't back it up — and the paragraph backs it up by listing the specific tools Alex has actually used. The paragraph also shows what Alex is *not* doing (e.g., not lumping a guardrailed builder in with raw codegen tools), which is harder to fake than a list of names.

**Hard rules:**
- **Name 3-5 reference points** at minimum: people, publications, products on each side of the live tension. Without specifics, the paragraph collapses to enthusiasm copy.
- **Include the live debate or tension** in the category, with Alex's actual position on it (one sentence). "Both have a place, here is the line I draw" beats "I love both."
- **Show usage, not just awareness.** Close with the products Alex has personally used on each side of the line ("agentic coding assistants on the codegen side; self-hosted no-code automation on the no-code side").
- **No "passionate about" or "excited about".** AI-writing tells, recruiter-flagged. Use verbs of practice: follow, use, watch, run.
- **Do not write this paragraph for categories Alex is *not* an insider in.** Faked fluency reads worse than no paragraph at all. If the target is in a category Alex only knows surface-deep (e.g., on-chain dev tooling, biotech informatics, climate-tech hardware), drop the pattern.

**HTML pattern:** plain `<p class="body">…</p>` (no special block).

**Canonical example:** Lumen AI letter, "On no-code specifically: I am a heavy user of self-hosted no-code automation…" through "…where each tool earns its keep."

## Common hook starters (pick one, adapt)

> **Prefer Personality Hooks from the research note (Step 0 in the tailoring checklist) when available.** They're calibrated to specific evidence about this company (founder backstory, manifesto line, customer overlap) and outperform any templated starter. The starters below are fallbacks for when Step 0 produced no high-authenticity hooks, or when the company genuinely has nothing distinctive to anchor on.

- "The [specific product / program / ritual] is what got me into this email." (concrete, low-risk)
- "I read [specific post / changelog / launch] last week and it lines up with what I've been running at Northwind Interactive for [X] years."
- "[Company] is one of a handful of teams where [specific thread] is treated as product work, not content support, which is the frame I've wanted to write a cover letter toward for a while."
- "I applied because [one concrete reason], and because the [community / creator / events] side of [Company] looks like the natural next surface for the program I've been running."

## Common transition threads (pick one, adapt)

- **Community intelligence:** weekly priority-tiered creator-content reports + daily sentiment digest to the team's chat, in production 6+ months with zero manual intervention.
- **Creator programs:** Creator Ambassador Program, tier-based, tracked output and rewards across full EU scope.
- **Cross-regional ops:** global IP crossover events (two major entertainment franchises) coordinated with HQ and licensor teams under strict IP restrictions.
- **AI workflow centralization:** shared knowledge base, 15+ role-specific assistants, reusable team prompts, Human-in-the-Loop safety protocols, recovering ~8 hours/week.
- **Event production:** on-site at SPL world finals, end-to-end from VIP creator care to live social activations across 6 platforms.

Happy to share specifics on any of the above, or to walk through how the Northwind scope translates to Lumen AI's community and creator surface. Open to a short call whenever suits you.
